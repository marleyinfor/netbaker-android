Por que isto?

A GPL obriga aos desenvolvedores distribuírem o código-fonte de seus produtos de software. A intenção é boa, mas representa um problema para companhias de software
e para desenvolvedores independentes: distribuir o código-fonte de seus produtos. Nós pensamos que o software proprietário é uma forma legal de distribuir software, 
sendo um direito de seu autor.

A LGPL pode ser a melhor escolha para preservar os nossos direitos e lhe dar a liberdade de criar novos produtos.

A licença de uso deste software é BASEADA na LGPL, mas possui alguns aditivos que você DEVE seguir.

Em caso de dúvidas, por favor contate os autores.

  Isto é o que você pode fazer com este software:

 3) Você pode criar um outro software que usa este software - USO SIMPLES;

 Preste atenção para a diferença entre TRABALHO DERIVADO e USO SIMPLES. Nós consideramos TRABALHO DERIVADO, um software
 que você criou BASEADO neste software. Isto sinifica:
 - Criar uma atualização deste software;
 - Criar outra versão deste software (para outra plataforma, por exemplo);
 
 "TRABALHO DERIVADO" significa que ESTE software é responsável pela maioria da funcionalidade do software derivado, não 
 importando se você utilizou o código-fonte ou uma versão compilada deste software. 
 
 "Uso simples" significa que você está apenas utilizando este software como parte da funcionalidade do seu próprio software. 
 Isto significa:
 - Criar um novo produto que use este software para compartilhar informação;
 
 Nós consideramos como "Uso simples" se este software não é a função principal do seu novo produto. E você pode fazer uso
 simples da versão compilada ou do código-fonte.
